Description

JXTAServices contains the services JXTASpool and JXTARendezvous. JXTAServices is part of JAME. JAME is an application for creating fractals and other graphics artifacts. JAME has a real-time, multi-threaded graphics engine with layers, filters, effects, a lot of formulas, and parameters. JAME supports operations such as continuous zoom of Mandelbrot and Julia sets, rotation and color shifting. JAME has also advanced features such as networked computation and scripting. You can take photos of your fractals or record movies from your exploration paths. Photos and movies are stored in a database to be rendered in high resolution and exported to a file. JAME can be extended by plug-ins which provide fractals, formulas, filters, effects and more.

How to install the services

Unzip the archive in your preferred location. JXTA services require Java SE version 1.6 or later for Windows and Linux, and version 1.5 or later for Mac OS X. You can download latest Java SE for Linux and Windows from http://java.sun.com/javase/downloads, and for Mac from http://developer.apple.com/java/download. JXTA services have been tested successfully on Ubuntu 9.10, Debian GNU/Linux 5.0, Windows XP (service pack 3), Windows Vista (service pack 2), Mac OS X 10.5 (Leopard) and Mac OS X 10.6 (Snow Leopard). For any problems, contact the author.

How to run the services

Click on the service icon to run JXTASpool. Alternatively, you can run the service from the command line with the command:

java -Xms128M -Xmx512M -Dosgi.install.area=. -Dosgi.configuration.area=configuration/JXTASpool -jar plugins/org.eclipse.equinox.launcher_<bundle-version>.jar -application net.sf.jame.service.jxta.spool

Click on the service icon to run JXTARendezvous. Alternatively, you can run the service from the command line with the command:

java -Xms128M -Xmx512M -Dosgi.install.area=. -Dosgi.configuration.area=configuration/JXTARendezvous -jar plugins/org.eclipse.equinox.launcher_<bundle-version>.jar -application net.sf.jame.service.jxta.rendezvous

JXTA services support custom properties which can be used to change the default behavior. The supported properties are listed on the table:

PropertyDescriptionExampleDefault value
debugLevelSets the log level-DdebugLevel=DEBUGINFO
appendersSets the appenders list-Dappenders=console,filefile
jame.maxJobCountChanges the maximum number of active jobs-Djame.maxJobCount=105
jame.workspaceChanges the workspace path (1)-Djame.workspace=/home/username/workspace${user.home}/JXTA<service-name>-workspace
jame.jxta.multicastEnables/disables multicast mode of JXTA node-Djame.jxta.multicast=truefalse
jame.jxta.relay.seedsChanges the seeds file of JXTA relay node-Djame.jxta.relay.seeds=/home/username/seeds.txt${user.home}/seeds.txt
jame.jxta.rendezvous.seedsChanges the seeds file of JXTA rendezvous node-Djame.jxta.rendezvous.seeds=/home/username/seeds.txt${user.home}/seeds.txt

(1) The workspace path is saved in the file ${user.home}/JAME.properties.

How to build the services

Download Eclipse SDK 3.5 or later from http://www.eclipse.org/downloads. Download the source code of JAME from the mercurial repository or download the latest snapshot from file releases. Unzip and run Eclipse, then import the main bundles ("Import existing projects on workspace"). Open the file JXTA<service-name>.product which is located in bundle net.sf.jame.service.jxta.product. Run the export wizard ("Eclipse product export wizard") and select the destination folder where you want to package the service. The wizard will create the executable and the folders features, plugins, and configuration. The generated launcher for Mac OS X has a bug which requires to replace the executable JXTA<service-name>.app/Contents/MacOS/JXTA<service-name> and the related library eclipse_<version>.so in bundle plugins/org.eclipse.equinox.launcher.cocoa.macosx.<arch>_<bundle-version> with the precompiled files which are located in folder patches/macosx (copy the files for your architecture).
